# GEMINI Project Instructions

## Project Overview
This is a **Next.js** project named `my-profile`. It appears to be a personal profile or portfolio site currently in its initial scaffolding phase (generated by `create-next-app`).

### Key Technologies
- **Framework:** Next.js (App Router)
- **Library:** React 19
- **Styling:** Tailwind CSS v4 (using PostCSS)
- **Language:** TypeScript
- **Fonts:** Geist Sans & Geist Mono (via `next/font/google`)

## Directory Structure
The main application is located in the `my-profile/` directory.

- `my-profile/app/`: Contains the application routes, layouts, and global styles.
- `my-profile/public/`: Static assets (SVG logos, favicons, etc.).
- `my-profile/next.config.ts`: Next.js configuration.
- `my-profile/tsconfig.json`: TypeScript configuration.

## Building and Running
All commands should be executed from within the `my-profile/` directory.

### Development
Starts the development server on `http://localhost:3000`:
```bash
npm run dev
```

### Build
Builds the application for production:
```bash
npm run build
```

### Production
Starts the production server after building:
```bash
npm run start
```

### Linting
Runs ESLint to check for code quality issues:
```bash
npm run lint
```

## Development Conventions
- **App Router:** Use the `app` directory for routing.
- **Components:** Favor Server Components by default; use `'use client'` only when necessary for interactivity.
- **Styling:** Use Tailwind CSS utility classes for styling. Global styles are managed in `app/globals.css`.
- **Types:** Strictly use TypeScript for all component and utility definitions.
- **Assets:** Place images and static files in the `public/` directory and use the `next/image` component for optimization.
